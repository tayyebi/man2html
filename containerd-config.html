NAME
       containerd-config - information on the containerd config

SYNOPSIS
       containerd config [command]

DESCRIPTION
       The  containerd config command has one subcommand, named default, which will display on standard output the default contain‐
       erd config for this version of the containerd daemon.

       This output can be piped to a containerd-config.toml(5) file and placed in /etc/containerd to be used as  the  configuration
       for  containerd on daemon startup. The configuration can be placed in any filesystem location and used with the --config op‐
       tion to the containerd daemon as well.

       See containerd-config.toml(5) for more information on the containerd configuration options.

OPTIONS
       default
              This subcommand will output the TOML formatted containerd configuration to standard output
